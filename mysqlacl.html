Fundamental Access Control

Pada keamanan informasi, kontrol akses sangat penting untuk memastikan kerahasiaan, integritas dan ketersediaan (CIA). Mengontrol siapa saja yang bisa mengakses ke sistem dan luasnya akses yang bisa dimiliki oleh pengguna dan sangat penting untuk memastikan dan mengamankan sistem dan data.

Contohnya seperti user root memberikan atau membuat user yang bisa mengakses ke database kepada host yang diizinkan dan hak akses yang diberikan.

Dengan perintah sebagai berikut :

CREATE USER player1@192.168.100.12 IDENTIFIED BY 'player1';
GRANT ALL ON MovieIndustry9.* TO player1@192.168.100.12;

Untuk pembuktian bahwa user tersebut hanya bisa mengkakses database yang diberikan hak akses saja, berikut gambar yang menampilkan daftar database pada user root, dan database pada player1

Adapun percobaannya seperti berikut:

Karena kita memberikan semua hak akses kepada player1, jadi untuk perintah create,read,update, dan delete bisa dijalankan oleh player1 .

Mandatory Access Control

Pada kemanan komputer Mandatory (wajib) Access Control (MAC) merupakan jenis akses kontrol dimana sistem operasi atau pada database yang membatasi kemampuan subjek atau inisiator (user) untuk mengakses atau pada umumnya melakukan operasi pada objek atau target biasanya pada database objeknya adalah tabel, kolom, tampilan dan lainnya. Pada dasarnya keamanan ini dikendalikan secara terpusat oleh administrator yang mengelola keamanan, jadi user lain tidak memiliki kemampuan untuk mengganti kebijakan.

Contohnya user yang tidak bisa mengakses database atau table yang telah ditentukan. Hanya root user yang bisa mengganti kebijakan.

Adapun perintahnya sebagai berikut.

CREATE USER player2@192.168.100.12 IDENTIFIED BY 'player2';
GRANT SELECT ON MovieIndustry9.* TO player2@192.168.100.12;
CREATE USER player3@192.168.100.12 IDENTIFIED BY 'player3';
GRANT SELECT ON MovieIndustry9.(satu table) TO player3@192.168.100.12;

Dalam hal ini player2 hanya bisa menggunakan perintah SELECT pada setiap tabel di database MovieIndustry9 dan player3 hanya bisa untuk menggunakan perintah select atau read pada tabel Movie di database MovieIndustry9.

Pada player2 saya akan memberikan beberapa perintah, seperti :

1. Select

Hal ini bisa dilakukan karena hak aksesnya adalah select, jika saya menggunakan perintah create maka tidak akan bisa dilakukan, seperti contoh pada dibawah ini.

2. Create

Hal ini tidak bisa dilakukan dikarenakan player2 tidak memiliki izin untuk melakukan hak akses create

Untuk player3 ini hanya bisa mengakses tabel Movie.

Discreationary Access Control

Kontrol akses diskresi (DAC) merupakan jenis kontrol akses yang ditentukan oleh Trusted Computer System Evaluation Criteria (TCSEC) sebagai sarana untuk membatasi akses ke objek berdasarkan identitas subjek atau kelompok mereka, dengan arti subjek dengan akses tertentu bisa meneruskan izin yang diberikan dan bisa meneruskan ke subjek lain ( kecuali ditahan oleh MAC).

Contoh ada 3 user dimana user 1 hanya untuk CRU user 2 CRUD player3 hanya R. di mana CRUD artinya Create, Read, Update, dan Delete.

Adapun perintahnya sebagai berikut.

CREATE USER player1@192.168.100.12 IDENTIFIED BY 'player1';
GRANT CREATE,INSERT,SELECT,UPDATE, ON ujicoba.* TO player1@192.168.100.12;
CREATE USER player2@192.168.100.12 IDENTIFIED BY 'player2';
GRANT CREATE,INSERT,SELECT,UPDATE, ON ujicoba.* TO player2@192.168.100.12;
CREATE USER player3@192.168.100.12 IDENTIFIED BY 'player3';
GRANT CREATE,INSERT,SELECT,UPDATE, ON ujicoba.* TO player3@192.168.100.12;
